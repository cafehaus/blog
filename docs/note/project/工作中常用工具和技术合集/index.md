## 前端团队协作
* z-index 规范
* 公用组件/样式/方法
* 公用的业务组件都喜欢直接复制出来改下，后期维护成本大，也容易出问题
* 尽量不要一次 commit 所有文件，跟模块化开发一样的，拆分成细小的修改，然后每一个点一次 commit
